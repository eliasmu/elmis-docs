Chapter 2. Report and Requisition

Report and Requisitions are the core practice of any Pull Logistics system. In a pull supply chain, the movement of stock is driven by request and reporting submitted by the pulling logistics unit. In the E-LMIS, this logistics unit is the service delivery unit (the Health Facility). The E-LMIS is designed to be configurable for different countries and different program implementations. To Acheive this, the E-LMIS has features to configure the report and requision form.

Note

Report and Requsition is also known as "R&R"

To configure the R&R Template for a given program area, please go to Administration -> Configure -> R&R Template

Figure 2.1. R&R Template Configuration

From the screen that was displayed, please select a program that you would like to customize.

A program could be configured as a "pull logistics program" or a "push logistics" program. The fundamental difference between these two types of programs is that pull program functions by the Service Delivery Unit (SDUs) sending a reqeust and requisition form. On the other hand, in a push program, SDUs are expected to be fulfilled by the Warehouses based on Formula's such as demographics based estimate instead of the requests from the service Delivery Units.

Note

Only "pull" programs are displayed in the E-LMIS R&R Template configuraiton page.

Figure 2.2. R&R Template Configuration Detail

The R&R Template Configuration detail page allows you to decide which columns are applicable for a specific program. You can also use this page to decide on and arrange the order of the Report and Requisition columns.
