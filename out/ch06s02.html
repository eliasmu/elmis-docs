Rest API

Restful web services are used in to facilitate interaction between different systems. The E-LMIS have a number of Rest API web services to allow you to perform different automated functions.

To use the Rest APIs an application needs to be able to authenticate with the system. Any user name and password in the system could be used to authenticate for the restful web services. However, the Web APIs will only make data available based on the permissions granted to the user.

There are two different categories of Rest APIs in the E-LMIS

Lookup Web APIs

Lookups are Web APIs that are used to list shared data elements from the E-LMIS.

A Swagger API Documentation for all of the web services could be found in the e-LMIS. To access the Swagger UI please go to http://path-to-your-server/public/api-docs/index.html.

Note

The API Documentations should be available to you only after you logged in to the E-LMIS with a valid user name and password in the same browser session.

Work-flow Web APIs

Work-Flow Web APIs can be used to post information from other systems. A good example of such a Rest API is the API that could be used to submit Requisitions from SDP version of the E-LMIS

Workflow Web APIs could be used submit data to the E-LMIS. There are two Key areas where the E-LMIS is able to accept data from other systems. 1) R&R submission, and 2) Proof of Delivery Submission. Any application built using modern programming platforms will be able to submit R&R or PODs using these web apis. The detail specification of these API end points could be found at the Swagger documentations.

Figure 6.3. Workflow Web APIs

Reporting Web APIs

Reporting Web APIs could be used to query the E-LMIS for a period specific report. An example of such a reporting Rest API is Stock Status web API.
